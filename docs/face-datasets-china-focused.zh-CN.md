# 适合 SR Data Maker 的中国人脸数据集整理

[English Version](face-datasets-china-focused.md)

这份文档只整理对本仓库真正有工程价值的人脸数据集，重点放在“中国人脸占比高”或者“明确以中国/东亚人脸为主”的数据源，服务于以下场景：

- 人脸超分训练数据生成
- 人脸修复 teacher 模型推理
- 退化生成
- YAML 数据源配置选型
- 评测集与对比集整理

## 推荐结论

| 数据集 | 中国/东亚人脸相关性 | 获取方式 | 更适合在本工程里做什么 | 推荐级别 |
| --- | --- | --- | --- | --- |
| `SCUT-FBP5500` | 强，明确包含 `2000 Asian females + 2000 Asian males` | 公开学术发布 | 作为主 HR 人脸来源、对齐裁脸、退化生成、teacher 推理 | 很高 |
| `Tsinghua-FED` | 强，明确为中国人脸 | 论文/学术获取 | 作为高质量 HR 参考集、验证集、展示样例集 | 很高 |
| `CAS-PEAL-R1` | 很强，明确是 Chinese face database | 官网申请/研究用途 | 做姿态、光照、遮挡等鲁棒性评测 | 高 |
| `SCUT-FBP` | 强，亚洲女性人脸 | 论文公开 | 作为小规模干净补充集、流程联调集 | 中高 |
| `SZU-EmoDage` | 中，更适合作为合成增强来源 | 论文/OSF | 做分布补充和增强，不建议做主真值 | 中 |
| `DataTang` 商业数据 | 中到高，取决于具体 SKU | 商业采购 | 做后续规模扩充 | 有预算时高 |

## 逐项说明

### `SCUT-FBP5500`

- 这是最适合先接入本工程的公开数据集之一。
- 优点是人脸比较规整，适合正脸为主的人脸超分流程，也可以直接拿来做 HR 原图池。
- 很适合接到你现在的生成链里：先做人脸对齐或裁脸，再做合成退化，或者直接跑人脸 teacher 模型做参考输出。
- 局限是野外复杂场景不够多，不能单靠它覆盖全部真实分布。
- 来源：
  - [SCUT-FBP5500 GitHub 发布页](https://github.com/HCIILAB/SCUT-FBP5500-Database-Release)
  - [SCUT-FBP5500 论文](https://arxiv.org/abs/1801.06345)

### `Tsinghua-FED`

- 这个数据集的价值不在规模，而在图像质量。
- 很适合拿来做高质量 HR teacher 参考图、小规模高标准验证集、不同人脸超分模型的可视化对比样例。
- 不足是样本量不大，而且更偏学术获取方式。
- 来源：
  - [Tsinghua-FED 论文](https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0231304)

### `CAS-PEAL-R1`

- 这是非常值得纳入项目的数据集，但用途要放对。
- 它更适合做姿态鲁棒性测试、光照变化测试、遮挡情况下的人脸处理评测。
- 不建议把它当成唯一主训练集，因为它的采集环境比较受控。
- 来源：
  - [CAS-PEAL-R1 官网](https://www.jdl.link/peal/home.htm)
  - [申请信息页](https://www.jdl.link/peal/Request%20Information.htm)

### `SCUT-FBP`

- 规模小，但很干净。
- 它很适合在工程里做这些事：跑通第一版人脸数据处理流程、验证 YAML 配置是否正确、做 smoke test。
- 来源：
  - [SCUT-FBP 论文](https://arxiv.org/abs/1511.02459)

### `SZU-EmoDage`

- 这个更建议放在“增强数据源”而不是“核心真值数据源”位置。
- 如果你的目标是补充分布、表情变化、合成扩展，它可以作为辅助；如果目标是构造高质量 HR 对，应优先真实拍摄数据。
- 来源：
  - [元数据记录](https://d-nb.info/1321435401/34)
  - [OSF 项目页](https://osf.io/7a5fs/)

### `DataTang` 商业数据

- 如果后面你要做真正大规模的数据生产，这一类商业数据很现实。
- 适合在公开数据已经不够用、需要更大规模的人脸数量、需要更丰富姿态或拍摄条件时考虑。
- 但要注意：商品页写“东亚/黄种人脸”不完全等于“中国人脸”，采购前要确认样本成分、授权条款、是否允许模型训练。
- 来源：
  - [DataTang 数据集 173](https://datatang.com/dataset/173)
  - [DataTang 数据集 1199](https://datatang.com/dataset/1199)

## 对本工程的推荐接入顺序

建议按下面顺序使用：

1. `SCUT-FBP5500 + SCUT-FBP`
2. `Tsinghua-FED`
3. `CAS-PEAL-R1`
4. 商业中文/东亚人脸数据

理由很直接：

- 第一阶段先把数据生成链稳定跑通
- 第二阶段提高 HR 参考质量
- 第三阶段做更严格的鲁棒性验证
- 第四阶段再考虑规模扩张

## 一个现实判断

目前没有一个“规模足够大、完全公开、明确中国人脸占多数、又天然适合超分训练”的完美数据集。

如果这个工程后面要走到真正可训练、可扩展、可持续产出训练对的阶段，比较现实的数据策略通常会是：

- 公开学术数据集做种子
- 自采或合作数据做补充
- 商业授权数据做规模扩张
